# 矿井危险区域人员闯入监控报警系统 - 前后端开发计划

## 一、项目概述

### 1.1 项目背景
本项目旨在构建一个基于YOLO目标检测算法的矿井危险区域人员闯入智能监控报警系统，实现危险区域人员入侵的自动化、实时化、可视化监控。

### 1.2 核心目标
- 实时检测：从视频帧采集到报警触发的总延迟≤500ms
- 精准识别：正常光照环境下人员检测准确率≥95%，恶劣环境≥88%
- 多维报警：分级报警与多渠道通知
- 全程追溯：支持实时监控、历史数据回溯、操作日志记录
- 稳定运行：支持7×24小时连续工作

## 七、技术难点与解决方案

### 7.1 视频流低延迟处理

**难点：**
- RTSP流解码延迟
- 多路视频并发处理
- 网络不稳定导致卡顿

**解决方案：**
1. 使用PyAV替代OpenCV解码，延迟降低30%
2. 采用异步IO处理多路视频流
3. 实现断线重连与帧缓存机制
4. RTSP强制使用TCP传输避免丢包

### 7.2 目标检测实时性

**难点：**
- GPU资源有限
- 恶劣环境检测精度低
- 多路视频并行推理

**解决方案：**
1. 采用可配置抽帧策略（1-10帧/秒）
2. 使用YOLOv8-S轻量模型
3. 批量推理优化GPU利用率
4. 针对矿井场景微调模型

### 7.3 入侵判定准确性

**难点：**
- 检测框抖动导致误判
- 人员短暂穿越误报
- 多人场景复杂判定

**解决方案：**
1. 采用中心点法表征位置
2. 引入停留阈值过滤短暂穿越
3. 使用目标追踪保持ID连续性
4. 多边形区域精准判定

### 7.4 前端实时性展示

**难点：**
- 多路视频同时展示
- WebSocket数据量大
- Canvas绘制性能

**解决方案：**
1. 采用分屏布局，按需加载
2. WebSocket数据压缩传输
3. 使用requestAnimationFrame优化绘制
4. 实现虚拟滚动优化列表

## 八、部署架构

### 8.1 Docker Compose配置

```yaml
version: '3.8'

services:
  # 后端服务
  backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data
    environment:
      - DATABASE_URL=mysql://root:password@db:3306/mine_safety
      - REDIS_URL=redis://redis:6379
    depends_on:
      - db
      - redis
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]

  # 前端服务
  frontend:
    build: ./frontend
    ports:
      - "80:80"
    depends_on:
      - backend

  # MySQL数据库
  db:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=password
      - MYSQL_DATABASE=mine_safety
    volumes:
      - mysql_data:/var/lib/mysql

  # Redis缓存
  redis:
    image: redis:7-alpine
    volumes:
      - redis_data:/data

volumes:
  mysql_data:
  redis_data:
```

### 8.2 Nginx配置

```nginx
server {
    listen 80;
    
    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /index.html;
    }
    
    # 后端API代理
    location /api {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    # WebSocket代理
    location /ws {
        proxy_pass http://backend:8000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```
